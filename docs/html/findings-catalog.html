Findings catalog#

Every finding the tool can emit has a stable ID (category.name). These IDs are a contract: they are stable across versions, appear in JSON output, and are what you reference in rules:/ignore.rules (Configuration).

Severities: critical, warning, info. Thresholds come from configuration; the triggers below describe the default behavior.

server#

IDSeverityTrigger
server.version_outdatedwarningmajor version < 7
server.recent_restartinfouptime < 1h
server.role_unexpectedinforole differs from --expect-role
server.high_client_countwarningconnected clients > 80% of maxclients
server.blocked_clientswarningblocked clients ≥ threshold
