Commands#

Every subcommand. All commands that touch Redis accept the connection flags and --config.

CommandPurpose
analyzeFull diagnostic run (the primary command).
scan-keysKeyspace + prefix report only.
inspect-streamDeep single-stream diagnostic.
inspect-clientsClient analysis only.
config-checkConfig + persistence + replication risk only.
analyze-sentinelSentinel topology diagnostic.
analyze-clusterCluster diagnostic.
reportRe-render a saved JSON report in another format.
diffDiff two saved JSON reports.
tuiInteractive terminal UI — see the TUI guide.
serveWeb GUI + JSON API — see the GUI guide.
versionPrint the version.

Connection flags#

Accepted by every command that connects to Redis:

<redis-url>                      redis://, rediss://, or unix:// (positional)
--host --port --db --username --password
--tls --tls-ca-cert --tls-cert --tls-key
--socket-timeout (default 5)     --connect-timeout (default 5)
--config <file>

A <redis-url> and explicit flags can be combined; flags override the URL. Passwords also come from REDIS_DOCTOR_PASSWORD. The password is never printed — see Safety.

Common output flags#

Where applicable: --format terminal|markdown|json, --output <path>, --fail-on none|warning|critical, --no-fail. See Output & exit codes.
